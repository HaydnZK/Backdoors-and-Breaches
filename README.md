# Backdoors & Breaches Scenarios
Welcome to the repository for my Backdoors & Breaches Incident Response Tabletop scenarios. This repo contains custom, modular scenarios designed to run tabletop exercises with your security team or community. Each scenario mimics the unpredictable flow of a live security incident, complete with attack paths, unexpected event escalations, and tactical options like calling external consultants, network isolation, or handling executive crisis management.

The rules are built to bridge the gap between abstract security concepts and hands-on analyst decisions. Every scenario includes explicit guidelines for handling failed rolls, categorizing them across personnel bottlenecks, technological failures, political friction, or direct financial constraints.

---

## Scenarios
* **[Cloud Chaos: The Ghost Ledger](https://github.com/HaydnZK/Backdoors-and-Breaches/tree/main/1.%20Cloud%20Chaos)**
An enterprise incident response simulation focusing on cloud identity provider exploitation and high-stakes data risk. The scenario starts with a leaked Personal Access Token in a public GitHub repository and tracks an attacker leveraging environment variables and web management portals to compromise a primary Domain Controller without triggering standard network alerts.
* **[The Trickle Effect](https://github.com/HaydnZK/Backdoors-and-Breaches/tree/main/2.%20The%20Trickle%20Effect)**
A classic modular worm infection that scales from a quiet human resources phishing loader into an operational crisis. This scenario forces defenders to deal with rapid internal lateral movement via legacy SMB vulnerabilities, hidden data exfiltration using high-volume DNS tunneling, and a chaotic multi-front distraction involving a live DDoS attack and a final wiper payload.
* **[Web of Lies](https://github.com/HaydnZK/Backdoors-and-Breaches/tree/main/3.%20Web%20of%20Lies)**
Inspired by real-world advanced persistent threat tactics, this scenario highlights the dangers of elite corporate social engineering and identity infrastructure compromise. The exercise tracks an off-hours voice phishing (vishing) attack on an IT helpdesk that leads to an unauthorized multi-factor authentication swap, OneDrive credential harvesting, and rogue cloud storage data staging.
* **[Salty Storm](https://github.com/HaydnZK/Backdoors-and-Breaches/tree/main/4.%20Salty%20Storm) (In Development)**
A complex scenario focusing on edge device exploitation and long-term, low-and-slow network persistence. This exercise will challenge teams to hunt for stealthy modifications to network access control lists, rogue routing tunnels, and unauthorized container execution deep within enterprise networking infrastructure.
